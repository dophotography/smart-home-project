Smart Home Core App Documentation

Imported Elements


          import { Lights, HeatingSystem, Lamp, TV, SmartHome } from './smart-elements.js';
          import { AudioPlayer } from './player.js';
          import { Weather } from './weather.js';
          import { Burger } from './burger.js';
          import { Modal } from './modal.js';
          import { abi } from './contract/abi.js';

The core app imports various elements for creating a smart home system.

  • Lights: Represents the smart lights element.
  • HeatingSystem: Represents the smart heating system element.
  • Lamp: Represents the smart lamp element.
  • TV: Represents the smart TV element.
  • SmartHome: Represents the overall smart home system.
  • AudioPlayer: Represents the music player element.
  • Weather: Represents the weather element.
  • Burger: Represents the burger menu element.
  • Modal: Represents the modal element for authentication.
  • abi: Represents the contract's ABI for interaction with the blockchain (You should provide this ABI as a value).

AuthContract Class Documentation

Constructor

constructor(abi)

Creates an instance of the AuthContract class.

  • abi (Array): The ABI (Application Binary Interface) of the Ethereum smart contract representing the authentication system.

Methods

checkMetaMask()

async checkMetaMask()

Checks if MetaMask is installed and available in the user's browser.

  • Returns:
    • true if MetaMask is installed and available.
    • false if MetaMask is not installed or not available.

Usage Example


          // Import the AuthContract class
          import { AuthContract } from './AuthContract.js';
          
          // Initialize the AuthContract with the ABI of the smart contract
          const authContract = new AuthContract(abi);
          
          // Connect the wallet and check login status
          async function login() {
            const isLoggedIn = await authContract.isUserLoggedIn();
            if (!isLoggedIn) {
              await authContract.connectWallet();
            }
          }
          
          // Change the house name
          async function changeHouseName(newName) {
            await authContract.changeHouseName(newName);
          }
          
          // Retrieve and display the current house name
          async function displayHouseName() {
            await authContract.showHouseName();
          }
          
          // Check if MetaMask is available
          const isMetaMaskInstalled = authContract.checkMetaMask();
          console.log('MetaMask Installed:', isMetaMaskInstalled);

Web3 Basics

Web3 is a JavaScript library used to interact with the Ethereum blockchain. It allows developers to build decentralized applications (DApps) and communicate with smart contracts on the blockchain.

  • ABI (Application Binary Interface): The ABI is a JSON representation of a smart contract's functions and events. It defines how to interact with a smart contract from outside the blockchain. It acts as an interface between your application and the smart contract, allowing you to call functions and listen for events.
  • Window Object: In the provided code, window is the global object in a web browser. It contains various properties and methods, including window.ethereum, which represents the user's Ethereum wallet (such as MetaMask) and provides access to the Ethereum blockchain.
  • Asynchronous Operations: Many functions in Web3, such as web3.eth.getAccounts() and contract.methods.isUserLoggedIn().call(), return Promises. Promises are used for handling asynchronous operations, such as making HTTP requests and waiting for responses from the blockchain. The async and await keywords are used to work with Promises in a more readable and synchronous-like manner.
